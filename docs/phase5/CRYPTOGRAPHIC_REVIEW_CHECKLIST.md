# Cryptographic Review Checklist

Reviewers should check:

1. correctness of expansion, addition, multiplication, and compaction invariants;
2. q-ary error accumulation and decoding failure rates;
3. sparse q-ary LPN sample counts from keys, ciphertexts, and FL deployments;
4. q-ary code/LPN compaction row-difference surfaces;
5. whether parameter sets avoid low-noise failure modes;
6. whether relation-resistant compaction truly avoids exploitable low-weight public relations;
7. whether demo/PQC wrapper boundaries are correctly marked;
8. whether claim language avoids certification/standardization overstatement.
