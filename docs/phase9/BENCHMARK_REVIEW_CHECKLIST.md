# Benchmark review checklist

Before using Phase 9 results in a manuscript, verify:

1. The same hardware and OS were used for all backends.
2. CPU scaling/turbo/background load were documented.
3. Each benchmark was repeated enough times to report mean and median.
4. Parameter/security descriptions were recorded for every backend.
5. SABLE toy/demo presets were not compared as production-secure parameter sets.
6. External library versions and commit hashes were recorded.
7. Missing external measurements were not inferred.
8. Source code for every baseline driver is included in supplementary materials or a public repository.
