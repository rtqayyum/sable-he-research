# External attack-review checklist

Reviewers should check:

1. Whether the sparse-LPN public rows match the assumptions used in estimates.
2. Whether q-ary LPN row-difference surfaces are correctly modeled.
3. Whether modern ISD algorithms reduce the claimed security margin.
4. Whether BKW/coded-BKW variants exploit the chosen q, noise, and sample counts.
5. Whether sparse row distributions or repeated public keys introduce additional structure.
6. Whether parameter-set tables should be rejected, increased, or narrowed to smaller workloads.
