# Phase 8 attack-review checklist

Reviewers should check:

1. exact public sample counts for each candidate;
2. sparse row distributions and collision probability;
3. q-ary BKW parameters, reduction levels, and sample complexity;
4. Prange/Stern/Dumer/BJMM/May--Ozerov-style estimates for the q-ary surfaces;
5. whether row-difference samples create a stronger attack surface;
6. deployment-scale input ciphertext accumulation;
7. whether any candidate should be removed before paper submission.
